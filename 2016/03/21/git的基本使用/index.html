4 bash

4.1 简单命令

命令格式：命令 -[参数] 文件夹名

命令：

pwd 查看当前目录的全名称

cd 切换目录

cd 目录名 当前目录下一级目录

cd ..切换到当前目录上一级目录

cd 子目录名 按tab键会出现智能提示，如果当前目录下只有一个子目录会自动补全，如果存在多个匹配，需输入多几个字母完成匹配

ls 获取当前目录下的子项

ls -l 以列表的形式展示了子项的详细信息

ls -a 在l的基础上增加两个信息

. 表示当前目录

.. 表示当前目录的上一级

ls -al 相当于 -a和-l的合集

ls -list 相当于 -l

ls可以查看任意文件夹的内容，比如以列表的形式展示 ls -l 文件夹名 或 ls -l ./文件夹名

6 版本控制

作用：用来管理代码， 他具有以下优势：

对于个人开发来说：

代码优化，代码更新；没有版本控制软件修改了哪些代码可能自己都不清楚了；

有了版本控制之后，可以将某个文件的新内容和旧内容进行对比，对比后就可以知道这两段代码之间有哪些区别。

对于团队开发来说：

一个项目肯定是由若干个开发人员一起开发的，不同人员开发的不同代码可能会发生冲突，使用版本控制软件可以实现协同开发，代码共享。

版本控制软件分类：可以完成个人开发的目的，但是无法进行团队开发

1.本地版本控制软件

2.集中式版本控制软件：SVN

既适用于个人开发也适用于团队开发

原理：首先每一个开发者要分配一个账号（账号是由svn的服务器管理员发放的），开发者利用账号开发，把功能开发好了之后，将功能提交到服务器中。

3.分布式版本控制软件: Git

原理：给每一个开发者分配一个账号，开发者完成某个开发任务的时候，可能首先将代码无数次的提交到本地仓库，最终完成某个开发任务的时候，将本地仓库中的代码一起提交到服务器中。

7 Git本地

组成：工作区和本地仓库，工作区就是编写代码的地方；本地仓库又分为暂存区和主仓库。

代码首先会放入暂存区，在暂存区的代码可以随时修改；修改完成后，会将代码直接提交到主仓库，主仓库的代码也可以修改，修改时代码会放入暂存区，随意进行修改代码。我们可以看到主仓库的代码修改，看不到暂存区的代码。

Git文件的集中状态：

在git管理的目录中，创建文件和修改文件或者删除文件时都不会被记录，要想文件被记录必须通过某个命令

未标记：未追踪（凭空创造，没有被git管理）

已提交：该文件已经被git管理起来，接下来对该文件进行相关操作就可以实现版本控制。

已修改：某个文件已经提交到暂存区了，如果对该文件进行修改，那么文件的状态就是已修改。

已暂存：某个文件已经被提交到暂存区了。

8.Git配置

git config –global user.name “自己的名字”con 设置全局所有仓库的名字

git config user.name “名字” 设置自己仓库的名字

git config –global user.email “自己的邮箱”

git init 对仓库进行初始化

git status 查看仓库中所有文件的状态，它会忽略到空目录，如果当前目录中有未追踪的内容，它也会提示相关目录

Untracked files 未追踪的文件

Change to be Commited 即将要被提交到主仓库的文件，也就是文件在暂存区。

modified 表示这是一个已修改的文件

git add 文件名 将未追踪的文件放入暂存区或也可以已修改的文件放入暂存区

git add * 将某个目录的全部内容放入暂存区

git commit 将暂存区的内容放入仓库

git commit -m 备注信息 如果不输入备注信息，会提示一个vi编辑器，每一个提交到本地仓库必须输入备注信息

git log 查看提交历史

显示结果有：

某一次提交的唯一标记

提交人的信息

提交的日期

git log –oneline 查单提交历史简略操作

git reflog 查看历史记录，包括每一次提交的信息，每一次版本回滚的操作

撤销操作：

git checkout 文件名：将一个文件已经放入到暂存区了，如果希望从暂存区会恢复修改前的状态，可以使用git checkout –文件名

将文件从暂存区中取出并替换本地文件

git rest HEAD文件名：将已经添加到暂存区的文件，并且文件没有被修改过，可以从暂存区中取出

将文件从暂存区中取出，但是并不用

恢复到以前提交的版本：从本地仓库中取出文件替换工作目录的文件

git reset –hard 版本号（版本号可以取前几位）回退之后，在该版本之后提交的版本就失效了，不可逆。

git config –list 查看配置信息
