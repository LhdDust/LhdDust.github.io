git的基本使用02

1 .gitignore

作用：编辑的不用提交的文件名

在根目录下创建.gitignore，添加不需要的提交的文件名

2. README.md

作用：是对项目的介绍和描述。

3.Repositories仓库

HTTPS：在http上做一些加密操作。

SSH：属于非对称性加密
对称性加密和非对称性加密
加密操作都是在数据传输的时候进行加密；
对称加密：数据通过某种算法加密，两台服务器必须知道该算法，通过算法进行解密，最后获取数据；加密的算法可能被窃取。

4. 提交本地仓库到远程服务器(远程服务器为空)

git push 远程服务器地址 master

注意点：第一次提交到.github的时候会提示你输入github的用户名或者密码。

5. 从服务器更新最新的代码

git pull –rebase <远程服务器地址 > master –rebase合并冲突

6. 保存服务器地址

因为每次提交都要重写仓库地址，所以把服务器地址添加到origin变量中

git remote add origin 服务器地址

删除变量

git remote remove 变量名

查看变量

git remote

7. 分支

分支使用策略：

每个开发人员进行开发一个新功能的时候，会分配一个独立分支，便于独立开发；

当功能完成时，将分支进行合并；将分支复制到主分支上，进行功能模块的整合。

8. 分支基本操作

git branch ：查看当前仓库的分支 *表示当前所处的分支

git branch 名称：创建一个新的分支，创建分支后，会拷贝当前分支的内容。

git checkout 分支名： 切换分支；切换分支后，不属于这个分支无法查看。

9 分支使用注意

1.在创建分支的时候，一定要对创建的分支所处于的那个分支，要在该分支稳定的时候才可以创建；在该分支的代码完全提交之后才能创建分支。

2.本地分支和远程分支是两个独立体

10 分支的合并和删除

git merge 分支名 把指定分支合并到当前分支
git branch -d 分支名 删除指定分支

11 项目克隆

git clone
– global本机只需要配置一次，本机的所有git仓库都是共享该信息的。

12 非对称性加密与对称性加密

13 git diff

a.比较工作目录和暂存区的代码比较
b.如果暂存区没有文件，则将工作区和最近一次提交的代码进行对比

14 从服务器中删除文件

git add 文件名 或者 git rm 文件名

git rm 文件名 操作是可逆的

直观的来讲，git rm 删除过的文件，执行 git commit -m “abc” 提交时，会自动将删除该文件的操作提交上去。

而用 rm 命令直接删除的文件，单纯执行 git commit -m “abc” 提交时，则不会将删除该文件的操作提交上去，需要在执行commit的时候，多加一个-a参数，
即rm删除后，需要使用git commit -am “abc”提交才会将删除文件的操作提交上去。
